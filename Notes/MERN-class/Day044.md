## 1. DOM Fundamentals

### What is the DOM?

The **DOM (Document Object Model)** is a tree-like representation of an HTML document created by the browser. Every HTML element becomes a **node** in this tree.

* JavaScript interacts with the DOM to update UI
* React does **not** directly manipulate the DOM
* React uses a **Virtual DOM** and updates only what changes

### Diffing Algorithm (React)

React compares the **previous Virtual DOM** with the **new Virtual DOM**.

* Finds the minimum number of changes ("diffing")
* Updates only changed nodes in the real DOM
* Makes UI updates fast and efficient

> Key idea: **UI updates are driven by state changes, not direct DOM manipulation**

## 2. `useRef` Hook

### What is `useRef`?

`useRef` creates a **mutable reference** that:

* Persists across re-renders
* Does **not** trigger re-render when updated
* Can point to DOM elements or store values

### Basic Example

```jsx
import { useRef } from "react"

const inputRef = useRef(null)

const handleClear = () => {
  if (inputRef.current) {
    inputRef.current.value = ""
  }
}

return (
  <>
    <input ref={inputRef} />
    <Button onClick={handleClear}>Clear All</Button>
  </>
)
```

### Key Notes

* `ref.current` holds the actual value
* Direct DOM access (escape hatch)
* **Survives re-rendering**

### `ref` vs `state` vs `props` (Interview Fundamental)

| Feature          | ref | state | props |
| ---------------- | --- | ----- | ----- |
| Causes re-render | ❌   | ✅     | ✅     |
| Mutable          | ✅   | ❌     | ❌     |
| Used for UI      | ❌   | ✅     | ✅     |
| Passed to child  | ❌   | ❌     | ✅     |
