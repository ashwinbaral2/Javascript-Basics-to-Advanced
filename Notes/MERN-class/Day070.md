# ⚙️ Backend Development

## 🚑 Ambulance Request Feature

A new feature was implemented to allow users to **request an ambulance through the platform**.

Before implementation, the following models were analyzed:

* **User Model**
* **Medical Infrastructure Model**
* **Ambulance Request Model**

The ambulance request connects these entities to create a **complete emergency transport request**.

### Data Relationship

```
User → Ambulance Request → Medical Infrastructure
```

Example structure:

```json
{
  "userId": "65f2a4c2",
  "hospitalId": "98d1b7a1",
  "pickupLocation": "Kathmandu",
  "destination": "City Hospital",
  "status": "pending",
  "requestedAt": "2026-03-12T08:30:00Z"
}
```

# 🔄 CRUD Operations Implemented

Full **CRUD functionality** was added for ambulance requests.

| Method     | Operation | Description                    |
| ---------- | --------- | ------------------------------ |
| **POST**   | Create    | Create a new ambulance request |
| **GET**    | Read      | Fetch request data             |
| **PATCH**  | Update    | Update specific fields         |
| **DELETE** | Delete    | Remove request                 |

### Example Endpoint

```bash
POST /api/ambulance-request
```

Example request:

```json
{
  "userId": "123",
  "pickupLocation": "Patient Home",
  "hospital": "City Hospital"
}
```

---

# ✏️ Partial Updates Using PATCH

The **PATCH method** was used to update only certain fields in a document instead of replacing the entire document.

### Example

Update request status:

```bash
PATCH /api/ambulance-request/:id
```

```json
{
  "status": "accepted"
}
```

### Advantages

* Faster database updates
* Reduced payload size
* Prevents overwriting unchanged data

# 🎨 Frontend Development

## Ambulance Request Confirmation UI

A simple UI component was added to confirm the ambulance request.

### Purpose

* Allows the user to **review their request**
* Confirms before sending the request to the backend

### UI Element

A **confirmation box** that triggers the request submission.

Example workflow:

```
User Clicks Request
      ↓
Confirmation Box Appears
      ↓
User Confirms Request
      ↓
API Call Sent to Backend
```
