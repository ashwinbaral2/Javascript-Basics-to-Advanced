# ⏱️ CRON Log Feature

A **cron job** is a **time-based task scheduler** used mainly in Unix/Linux systems.

It allows servers to **automatically execute scripts at specific times or intervals** without human intervention.

Behind the scenes, a background process called the **cron daemon (`crond`)** continuously checks the schedule and runs tasks when the time matches.

---

## 🧩 Cron Expression Format

A cron schedule consists of **five fields**:

```
Minute Hour DayOfMonth Month DayOfWeek
```

Example:

```
0 2 * * *
```

Meaning:

```
Run every day at 2:00 AM
```

---

* 📦 **Database backups**

* 📊 **Sending scheduled reports**
* 🔄 **Syncing external APIs**
* 🧾 **Log rotation**

# 📧 Nodemailer for Email System

A common way to send email in Node.js is using **Nodemailer**.

Install:

```bash
npm install nodemailer
```

---

## Example Email Sender

```javascript
import nodemailer from "nodemailer";

const sendMail = async ({ email, code }) => {
  try {
    const transporter = nodemailer.createTransport({
      service: "gmail",
      auth: {
        user: process.env.EMAIL_USER,
        pass: process.env.EMAIL_PASS
      }
    });

    const mailOptions = {
      from: process.env.EMAIL_USER,
      to: email,
      subject: "Your Medlink Verification Code",
      html: `
        <h2>Verify Your Account</h2>
        <p>Your verification code is:</p>
        <h1>${code}</h1>
        <p>This code expires in 1 hour.</p>
      `
    };

    await transporter.sendMail(mailOptions);

  } catch (error) {
    console.error("Email sending error:", error);
    throw new Error(error.message);
  }
};

export default sendMail;
```

## 🔐 Gmail App Password Setup

Because Gmail blocks normal password authentication, you must use an **App Password**.

Steps:

1. Enable **2-Step Verification** on Google account
2. Go to **Google App Passwords**
3. Generate password for:

```
App: Mail
Device: Other
```

4. Use it in `.env`

```
EMAIL_USER=your_email@gmail.com
EMAIL_PASS=generated_app_password
```

## Backend SMS Sender

```javascript
import twilio from "twilio";

const client = twilio(
  process.env.TWILIO_ACCOUNT_SID,
  process.env.TWILIO_AUTH_TOKEN
);

export const sendSMS = async (phone, message) => {
  try {
    await client.messages.create({
      body: message,
      from: process.env.TWILIO_PHONE,
      to: phone
    });

    console.log("SMS sent successfully");

  } catch (error) {
    console.error("SMS error:", error);
  }
};
```

## Example Admin API

```javascript
export const broadcastSMS = async (req, res) => {
  try {
    const { message } = req.body;

    const users = await User.find({ phone: { $exists: true } });

    for (const user of users) {
      await sendSMS(user.phone, message);
    }

    res.json({ success: true, message: "SMS sent to all users" });

  } catch (error) {
    res.status(500).json({ error: error.message });
  }
};
```

# ⚠️ Important Reality Check

Mass SMS systems must consider:

### 💰 Cost

SMS APIs charge **per message**.

Example:

```
1000 users = 1000 SMS
```

So:

```
Large user base = expensive
```

---

### ⏳ Rate Limits

Sending thousands of messages instantly may cause:

* API throttling
* message queue delays

Solutions:

* message queues
* background workers
* cron batches
