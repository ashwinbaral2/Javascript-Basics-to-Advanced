## 🌍 1️⃣ Loading Data Based on Location Coordinates

Today’s focus: **fetching data from the server/database using latitude & longitude** and displaying it dynamically on the client-side map.

### 🔹 Flow Overview

1. Client gets user location (or selected coordinates)
2. Coordinates sent to backend (`lat`, `lng`)
3. Backend queries database (MongoDB with geospatial support)
4. Filtered results returned
5. Data rendered on map

### 🔹 Backend Concept (Geo Query)

In MongoDB, this typically uses:

```js
{
  location: {
    $near: {
      $geometry: {
        type: "Point",
        coordinates: [lng, lat]
      },
      $maxDistance: 10000
    }
  }
}
```

👉 Requires `2dsphere` index on location field.

Traditional way. Efficient. Clean. Scalable.

## ⚛️ 3️⃣ Usage of `useEffect` with Map

`useEffect` is used to:

* Fetch data when component mounts
* Refetch when coordinates change
* Update markers dynamically

### Example Pattern

```js
useEffect(() => {
  const fetchData = async () => {
    const res = await fetch(`/api/medical?lat=${lat}&lng=${lng}`);
    const data = await res.json();
    setMedicalData(data);
  };

  if (lat && lng) {
    fetchData();
  }
}, [lat, lng]);
```

### Why dependency array matters?

If `[lat, lng]` changes → fetch again
If empty `[]` → only runs once

Get this wrong and:

* You either spam your API
* Or your UI never updates

Choose wisely.

## 🧩 4️⃣ Separating Hospitals, Blood Banks & Ambulances on Click

Instead of loading everything blindly, better UX is:

* Load all data initially
* Use filters/toggles on client side
* Show/hide markers by category

### Example State Logic

```js
const [filter, setFilter] = useState("all");
```

On button click:

```js
setFilter("hospital");
```

Rendering conditionally:

```js
medicalData
  .filter(item => filter === "all" || item.type === filter)
  .map(item => (
    <Marker key={item._id} position={[item.lat, item.lng]} />
  ));
```

Simple. Clean. Maintainable.
