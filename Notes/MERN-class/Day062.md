# 🗄 Database Discussion

## MongoDB vs SQL (Cost Perspective)

Concern raised:
> MongoDB (especially Atlas) can become expensive at scale.

### MongoDB (NoSQL)
- Flexible schema
- Fast iteration
- Good for evolving projects
- Scales horizontally
- Managed cloud (Atlas) may increase cost

### SQL (PostgreSQL / MySQL)
- Structured schema
- ACID guarantees
- Often cheaper at scale
- Better for relational-heavy systems
- Strong reporting capabilities

### Conclusion
Choose based on:
- Data structure complexity
- Budget
- Query requirements
- Future scalability needs

---

# ☁ MongoDB Atlas

Cloud-managed MongoDB solution.

Benefits:
- Automatic scaling
- Managed backups
- Built-in monitoring
- Easy integration

Risk:
- Pricing tiers increase with usage

# 🧱 Backend Folder Architecture

Recommended structure:

```bash

src/
│
├── index.js        → Entry point
├── db/             → Database connection
├── models/         → Mongoose models
├── routes/         → Express routes
├── utils/          → Helper functions
└── middleware/     → Auth & validations

```

## Benefits:
- Clean separation of concerns
- Easier testing
- Scalable architecture
- Maintainable codebase
