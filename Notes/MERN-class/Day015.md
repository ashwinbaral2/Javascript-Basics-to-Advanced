## ðŸ”¹ Optional Chaining `(?.)`

- **Purpose:** Safely access nested properties without running into `undefined` or `null` errors.  
- **Highly recommended** for interviews and clean code practices.

### âœ… Usage

1. **Accessing object properties**
```js
const user = { name: "Baral", address: { city: "Kathmandu" } };

// Traditional approach:
const city = user && user.address && user.address.city;

// Optional chaining:
const city = user?.address?.city; // "Kathmandu"
````

2. **Accessing array elements**

```js
const arr = [{ id: 1 }, { id: 2 }];

const secondId = arr?.[1]?.id; // 2
const thirdId = arr?.[2]?.id; // undefined, no error
```

### âš¡ Key Notes

* Stops runtime errors when a property in the chain is `null` or `undefined`.
* Can be combined with **function calls**:

```js
const result = obj?.method?.();
```

* Cleaner and safer than multiple `&&` checks.

---

> **Pro Tip:** Use optional chaining liberally in data-heavy applications like MERN stacks to prevent crashes from missing data.
