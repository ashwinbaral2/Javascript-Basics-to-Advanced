# Important React Hooks (Usage Table)

| Hook                      | Purpose                               | When to Use                                         | Key Characteristics                 | Example Use Case              |
| ------------------------- | ------------------------------------- | --------------------------------------------------- | ----------------------------------- | ----------------------------- |
| **`useState`**            | Manage component state                | When UI needs to update on data change              | Causes re-render when state changes | Form inputs, toggle buttons   |
| **`useEffect`**           | Handle side effects                   | API calls, subscriptions, timers                    | Runs after render                   | Fetching data from server     |
| **`useRef`**              | Store mutable value without re-render | Access DOM or persist values between renders        | Does **not trigger re-render**      | Focus input field             |
| **`useMemo`**             | Memoize expensive calculations        | When heavy computations should not run every render | Returns cached value                | Filtering large lists         |
| **`useCallback`**         | Memoize functions                     | Prevent unnecessary re-renders in child components  | Returns cached function             | Passing stable function props |

# 1️⃣ `useRef()` Hook

### Purpose

Stores a **mutable reference value** that persists across renders **without causing re-renders**.

### Syntax

```javascript
const ref = useRef(initialValue)
```

### Key Uses

| Use Case              | Explanation                        |
| --------------------- | ---------------------------------- |
| Access DOM elements   | Directly manipulate DOM            |
| Store previous values | Persist value across renders       |
| Store timers/IDs      | Avoid re-render when value changes |
| Avoid re-renders      | Mutable storage                    |

### Example

```javascript
import { useRef } from "react";

function InputFocus() {
  const inputRef = useRef(null);

  const focusInput = () => {
    inputRef.current.focus();
  };

  return (
    <>
      <input ref={inputRef} />
      <button onClick={focusInput}>Focus</button>
    </>
  );
}
```

### Important Points

* `.current` stores the value
* Changing it **does NOT re-render**
* Good for **DOM access**

# 2️⃣ `useMemo()` Hook

### Purpose

**Memoizes a computed value** to avoid expensive recalculations.

### Syntax

```javascript
const memoizedValue = useMemo(() => {
  return expensiveFunction(data);
}, [data]);
```

### Key Uses

| Use Case                 | Explanation                     |
| ------------------------ | ------------------------------- |
| Expensive calculations   | Avoid recalculating each render |
| Performance optimization | Cache results                   |
| Derived state            | Compute filtered/sorted data    |

### Example

```javascript
import { useMemo } from "react";

function Numbers({ numbers }) {

  const sortedNumbers = useMemo(() => {
    return numbers.sort((a, b) => a - b);
  }, [numbers]);

  return <div>{sortedNumbers.join(", ")}</div>;
}
```

### Important Points

* Runs **only when dependency changes**
* Improves performance
* Should **not be overused**

# 3️⃣ `useCallback()` Hook

### Purpose

Memoizes a **function** so it doesn't get recreated every render.

### Syntax

```javascript
const memoizedFunction = useCallback(() => {
  doSomething();
}, [dependencies]);
```

### Example

```javascript
const handleClick = useCallback(() => {
  console.log("Clicked");
}, []);
```

### Key Points

| Feature                   | Explanation                            |
| ------------------------- | -------------------------------------- |
| Prevents re-renders       | Useful for child components            |
| Returns memoized function | Similar to `useMemo` but for functions |

# 5️⃣ `useReducer()` Hook

### Purpose

Manage **complex state logic**.

### Syntax

```javascript
const [state, dispatch] = useReducer(reducer, initialState);
```

### Example

```javascript
function reducer(state, action) {
  switch(action.type) {
    case "increment":
      return {count: state.count + 1};
    default:
      return state;
  }
}
```

### When to Use

| Situation              | Reason                |
| ---------------------- | --------------------- |
| Multiple state updates | Cleaner than useState |
| Complex logic          | Redux-like pattern    |
| State transitions      | Controlled updates    |

# Quick Difference (Most Asked in Interviews)

| Hook          | Stores            | Causes Re-render?              |
| ------------- | ----------------- | ------------------------------ |
| `useState`    | State value       | Yes                            |
| `useRef`      | Mutable reference | No                             |
| `useMemo`     | Memoized value    | No (unless dependency changes) |
| `useCallback` | Memoized function | No                             |
