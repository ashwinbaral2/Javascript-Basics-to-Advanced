**ğŸ“Œ DEFINITION:**

JavaScript's behavior of moving declarations (not values) to the top before code execution.

**ğŸ“Œ KEY DIFFERENCES:**

- `var` is hoisted and initialized as `undefined`
- `let` and `const` are hoisted but **NOT initialized**

**âš ï¸ IMPORTANT:**

This is why accessing `let`/`const` before declaration causes ReferenceError.

```javascript

console.log(y); // ReferenceError (not initialized)

```

**ğŸ“Œ HOW IT WORKS:**
